## 0.1.60

A reliability release that broadens CaveCMS's database compatibility: it now
installs, updates, and runs cleanly on every supported MySQL and MariaDB
version — including the MariaDB 10.6 that ships with Ubuntu 22.04 LTS.

### Consistent across every database version

CaveCMS verifies, on every start, that your database matches the version of
the software you're running. That check used to read the database's own
description of its tables — but different MySQL and MariaDB versions describe
the *same* schema differently, so an install that started fine on one server
could refuse to start on another. The check now compares the set of applied
database updates instead, which is identical on every engine. The result: an
install that works on one server works on all of them, and updates apply the
same way everywhere.

### Correct optional-date handling on older MariaDB

On older MariaDB versions, several fields that are meant to be optional —
publish dates, account lock expiry, last-login time, and soft-delete markers —
were being stored as *required*. That could cause publishing a draft, clearing
a login lockout, or soft-deleting content to fail. These fields are now pinned
to the correct optional shape on every database version, finishing a repair
that an earlier release started for soft-delete markers.

Nothing to configure — updating applies everything automatically.
