## 0.1.44

CLI hotfix. Operators on slow connections were timing out, and an
operator reaching the password prompt saw the install exit
silently instead of advancing to the DB name + site URL prompts.

### Fixed

**`create-cavecms` exited silently at the password prompt.**
`askSecret()` was calling `process.stdin.pause()` after the
operator pressed Enter on the password. The shared readline
instance kept being used for the prompts that follow (DB name,
public site URL, port), but with stdin paused the next
`rl.question()` never received input — the event loop emptied,
Node exited cleanly mid-install, and the operator saw the prompt
walk just stop. Restoring raw mode + detaching our own listener
is sufficient; pausing was always wrong.

**Slow links timed out downloading the release zip.** `curl
--max-time 600` (10 minutes) on a ~70 MiB zip required ≥117 KB/s
sustained — fine for fibre, hostile on patchy mobile / overseas
links. Raised to `--max-time 1800` (30 min), added
`--connect-timeout 30`, plus 2 retries with a 5s delay to cover
transient TLS GOAWAY / TCP RSTs without bothering the operator.

**Unbounded manifest fetch could hang on a stalled TLS handshake.**
Native `fetch()` had no timeout. Added a 60-second
`AbortSignal.timeout` — enough for a slow link to deliver a few
KiB of JSON, short enough that a wedged connection fails loudly
instead of silently spinning.

### Tooling

`~/connect/connect` semver-comparison block for the CLI auto-
publish step used `return` at top-level `node -e`, which throws
`SyntaxError` — `is_newer` came back empty and the publish step
silently skipped. Replaced with `node -p` + an IIFE so the
comparison actually runs. (This is a Derrick-local tool fix; not
shipped in the customer release, but called out here because it
explains why `create-cavecms@latest` reported 0.1.8 on npm even
after the 0.1.43 release was published — the auto-publish never
fired. 0.1.44 publishes correctly.)

### Migration impact

None — CLI-only. No CaveCMS code changed. Existing installs are
unaffected; this release exists so the next operator who runs
`npx create-cavecms@latest` gets a CLI that walks the full prompt
sequence and tolerates a slow link.
