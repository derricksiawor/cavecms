## 0.1.4

Third (and final) fix-up in the 0.1.1 release-pipeline series. The
previous attempts (0.1.2 + 0.1.3) got mysql2 + drizzle-orm into the
staging tree as symlinks, but the **`zip -r` command on the macOS
build host was dereferencing them at archive time** — the unpacked
customer install still ended up with real top-level dirs missing the
sibling-dep chain.

### Fix

`build-zip.mjs`'s zip invocation now includes the `-y` flag, which
on BSD `zip` (macOS) preserves symbolic links instead of compressing
the file at the link's target. GNU `zip` on Linux preserves by
default; `-y` is a no-op there. Verified end-to-end locally:

```
unzip cavecms-0.1.4.zip -d /tmp/test-extract
ls -la /tmp/test-extract/cavecms-0.1.4/node_modules/mysql2
# → lrwxr-xr-x ... mysql2 -> .pnpm/mysql2@.../node_modules/mysql2

node -e "require('mysql2/promise')"   # ✓ resolves cleanly
```

### What this finally enables

- `npx create-cavecms` now completes through step 7 (PM2 start) on a
  fresh Linux box, including the migration step that died on every
  previous release.
- The in-app updater applying 0.1.4 onto a 0.1.3 install restores
  symlink integrity in the install dir (snapshot-based rollback +
  the same unpack path).

### Why this took three releases

Each fix was correct AND incomplete:

- 0.1.2 added the packages to `serverExternalPackages` (necessary —
  Next would otherwise inline them into webpack chunks, no on-disk
  presence).
- 0.1.3 switched the staging copy to `cp -a` (necessary — Node's
  `fs.cpSync` absolutifies symlinks to `/Users/<dev>/...` which
  breaks at customer extract time).
- 0.1.4 adds `zip -y` (necessary — the final hop where macOS' default
  zip behaviour silently undid all the prior preservation).

The whole chain has to preserve symlinks: pnpm install → Next standalone
→ cpSync → zip → unzip → Node resolver. Any link broken anywhere in
that chain falls back to dereferenced real dirs + sibling-dep loss.
0.1.4 closes the last gap.

### Nothing else changed

Same scope as 0.1.2 + 0.1.3 — pure release-pipeline correctness.
