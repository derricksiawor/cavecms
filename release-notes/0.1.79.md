## 0.1.79

Maintenance mode and the IP allow/block lists can now be enabled on a laptop / self-managed install.

### Fixed

**Maintenance mode was impossible to enable on a bare-node (laptop) install.** Turning on maintenance — or the IP allow/block lists — requires CaveCMS to know your own IP so it can keep you from locking yourself out. It reads that IP from the `X-Real-IP` header a reverse proxy sets. A laptop / self-managed install has no reverse proxy, so your IP came back undetectable and the dashboard refused with *"Your IP couldn't be detected; fix X-Real-IP forwarding before enabling maintenance."* On a laptop install the operator genuinely is local, so CaveCMS now resolves the loopback address there and the toggle works. Proxied installs (the systemd / PM2 / cPanel surfaces) are unchanged — they still require a real `X-Real-IP`, so the production trust model is intact.

### Migration impact

None. Runtime behaviour change only, scoped to the laptop surface. A laptop install that is later exposed publicly must forward `X-Real-IP` (the same requirement as any proxied install).
