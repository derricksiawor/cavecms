## 0.1.57

Recover an install from the command line when the dashboard isn't reachable.
A new `cavecms` command lets you update or roll back over SSH — for when
you're locked out of the admin login, or a bad release left the site unable
to load `/admin` at all.

### `cavecms update` and `cavecms rollback` from the shell

Until now, applying an update or rolling one back could only be done from the
**Settings → Updates** dashboard. If you couldn't reach the dashboard — a
forgotten password, an IP lockout, an expired session, or a release that broke
the app itself — there was no supported way to recover without hand-editing
files on the server.

Every install now ships a `cavecms` command you can run over SSH:

- `cavecms update` — fetch and apply the latest release (`--check` to see
  what's available without applying, `--force` to re-install the current
  version when a build is corrupted).
- `cavecms rollback` — restore the previous version from the most recent
  snapshot. This works offline and needs no network, so it's the fastest way
  back online after a bad release.
- `cavecms status` / `cavecms version` — see the running version and whether
  an update is available.

It applies the **same checks as the dashboard** — SHA-256 + Ed25519 signature
verification on every release, the same site-stays-online-until-verified flow,
and the same automatic rollback if a new version doesn't come up healthy. It
restarts your site the right way for how it's hosted (systemd, PM2, or
Passenger), and refuses to run two updates at once so a dashboard update and a
shell update can't collide.

### How to use it

Run it from your install directory (or pass `--dir /path/to/install`):

    cavecms update        # pull and apply the latest release
    cavecms rollback      # restore the previous version
    cavecms status        # what's running, and is an update available?

On servers where the app runs under a dedicated service user (the common
setup), run it as that user — `cavecms` tells you exactly how if you need to.
