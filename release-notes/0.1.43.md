## 0.1.43

Flushes the cover image up against the site header on every
industry template and paints the page eyebrow + title + body
(+ optional CTA) directly over the photo. Closes the gap that
sat between the header and the hero image, and the silent
"cover image, no copy" state every template inherited.

### Changed

**Cover image overlay text + flush hero on every template.** The
old pattern — `coverImage(...)` followed by a separate
`hero(...)` — produced two stacked sections with a visible gap
between them on every template page (home, sub-pages, contact,
press, etc.) and left the cover image as a silent photo with no
introduction. The new `heroCover(...)` helper:

- Uses a `none`-padded section so the image sits flush against
  the site header (no `py-*` gap at the top).
- Renders the eyebrow / title / body / CTA as an absolutely-
  positioned overlay block on the image itself, anchored to the
  bottom-left by default (9-point compass available via
  `overlayAlignment`).
- Ships two tone presets — `ivory` (default, for dark photos)
  and `obsidian` (for light photos).
- Defaults to a `gradient-bottom` overlay so the overlay copy
  reads on any image without darkening the whole photo flat.

Templates updated end-to-end: `hotel-solenne` (5 pages),
`anara-wellness` (4 pages), `grace-hill` (5 pages),
`harbor-and-lane` (4 pages), `ironclad-build` (4 pages),
`maison-brulee` (5 pages), `studio-verde` (4 pages), `velocity`
(4 pages). `default-welcome` has no cover image so is
unchanged.

### Schema

`lx_cover_image` block now accepts five new optional fields:
`eyebrow`, `title`, `body`, `cta { label, href }`,
`overlayAlignment`, `overlayTone`. Existing rows that don't set
them keep rendering as a pure cover image — backwards
compatible.

Section meta `padding` enum gains a `none` option (`Flush`)
that emits `py-0` — required for the flush-against-header
behaviour.

### CLI

`create-cavecms` jumps from `0.1.8` to `0.1.43` to track the
CaveCMS release version going forward. Behavior unchanged —
the CLI still downloads whatever the latest signed release zip
is. The version bump is cosmetic so `npx create-cavecms` and
the CaveCMS site you install both report the same number.

### Migration impact

None — no schema migration. The new fields on `lx_cover_image`
default to empty strings / null, so every existing cover image
block continues to render as a pure photo.
