## 0.1.85

Theme your project lead forms to any palette colour, drop in a full-bleed cover-image section in one click, plus a more responsive API Tokens page and a handful of fixes.

### New — paint the inquiry & brochure sections with your theme

The **inquiry** and **brochure** sections on a project page used to be locked to the cream look. Now each has a **Section background** option drawn straight from your theme palette — Cream, Ivory, Bone, Champagne, or Obsidian (dark). Pick one and the whole section re-skins together: heading, intro copy, the trust pills, the form card, and the inputs all shift to stay legible — including a dark, high-contrast input treatment when you choose Obsidian. It's brand-linked, so changing your palette under **Settings → Theme** re-skins these sections too. The default stays cream, identical to before, so nothing changes until you reach for it.

### New — one-click cover-image section

The block picker's "Insert section" menu gains a **Cover image** preset: a full-bleed section with a background photo, a dark gradient overlay for text legibility, and generous padding — ready for a heading and a button on top. Click it, pick your image, and you have a hero-style band without wiring up a section, background, and overlay by hand.

### Editor & polish

- The **Settings → API Tokens** page is now fully responsive — the token table and create form lay out cleanly on phones and tablets.

### Fixes

- Cleared two console hydration warnings — one from the site header's active-nav highlight, one from the editor's section-selection chrome — so the browser console stays clean on first paint.
- The standalone `cavecms rollback` command now restores your database alongside the code when a rollback crosses a migration, so a rolled-back site matches the version it rolled back to.
- Update errors now stay on screen long enough to read (toasts persist for 10 seconds) and surface the actual failure instead of a generic message.
- Uploading an image for a section or column **background** now shows it right away, instead of only appearing after a page refresh.

Updating applies everything automatically.
