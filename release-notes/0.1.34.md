## 0.1.34

In-app updater hotfix — start-standalone.mjs crashed on every update.

### Fixed

**`start-standalone.mjs` crashed at boot after every in-app update.**
The script copies `.next/static` and `public/` into the standalone
tree at startup. After the in-app updater's tarball extract,
`.next/standalone/.next/static` is already a symlink pointing at the
real `.next/static` (Next.js's standalone build ships them that way).
Node's `cpSync` refused to copy a directory onto its own symlink
target — `ERR_FS_CP_EINVAL: src and dest cannot be the same`. The
new process crashed before binding port; PM2's reload had nothing
healthy to serve; the orchestrator's step 6 verify saw "Connection
refused" for 60 iterations and rolled back. Every customer in-app
update of a CLI install ended in the rollback path.

`start-standalone.mjs` now compares `realpathSync(src)` against
`realpathSync(dst)` before the `cpSync`; identical → skip with a
"already linked" log line. If `dst` is a symlink to anywhere else,
unlink it first so the cpSync runs cleanly. The fresh-install path
(no symlink yet) still copies as before.

### Migration impact

Apply directly via the in-app updater from 0.1.16-0.1.33 — the new
`start-standalone.mjs` is extracted BEFORE PM2 reload runs the start
command, so the fix takes effect on the very first reload, not after
a reinstall.
