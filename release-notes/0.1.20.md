## 0.1.20

The big template release — eight industry-specific site starters land,
the install wizard gains a template-picker step, the branding step
gets a logo upload, and the admin sidebar now shows the running
version. Plus the orchestrator-helpers packaging fix that 0.1.19 was
meant to ship (0.1.19's zip never landed — scp broke mid-upload).

### Added

**Eight industry-specific site templates.** `lib/cms/siteTemplates/`
ships 9 templates (8 industry + 1 default-welcome) authored as full
SectionSpec trees — 1016 widgets across the registry, validated
end-to-end by `scripts/validate-site-templates.ts` (wired into
`prebuild`). The eight industry templates:

* Hôtel Solenne — hospitality
* Harbor & Lane — real estate
* Ironclad Build — construction
* Grace Hill — church
* Velocity — software
* Studio Verde — freelance creative
* Maison Brûlée — restaurant
* Anara Wellness — spa / wellness

Each ships Home / About / Services (or industry-equivalent) / Contact
+ branding defaults matched to the template's vibe. Default-welcome
is the operator's "I'll pick later" fallback — same content as the
0.1.x default install.

**Install wizard — template-picker step.** A new step lands between
Site and Branding. Visual tile grid of all 9 templates (selected
tile gets a copper ring; default-welcome is pre-selected so the
operator who skips ends up with the existing default behaviour, no
regression). Wizard total grows from 8 to 9 steps.

**Install wizard — logo upload.** The Branding step now accepts a
logo file. Upload goes to `POST /api/install/logo` which writes to
the uploads/originals tree with auto-variant generation (thumb / md /
lg via sharp), records the row in `media`, and stamps
`settings.site_header.logo_media_id`. The site header renders the
logo when set, falls back to text otherwise. DELETE handler reverts.
Both endpoints share the CSRF + install-token gates of the other
wizard endpoints.

**Admin sidebar shows the running version.** A muted `v0.1.20` chip
sits next to the "CaveCMS" wordmark in the desktop sidebar.
Bundled at build time from package.json, so it survives the standalone
bundle's typical package.json gaps. Operators confirm the running
version at a glance without navigating to Settings → Updates.

### Fixed

**Updater preflight no longer stalls on "scripts/lib/cavecms-update-helpers.sh
not found".** Originally meant for 0.1.19 but that release didn't
publish. The release-build step now copies `scripts/cavecms-update.sh`,
`scripts/cavecms-watchdog.sh`, and `scripts/lib/` into
`.next/standalone/scripts/` so the orchestrator can source its helpers
no matter which cwd it resolves from (Next.js's standalone tracer only
follows JS imports, not shell-sourced files).

### Migration impact

None for the codebase — all changes are additive. Existing installs
that already managed past 0.1.15's status-path bug keep working; the
new template + logo endpoints sit behind the install gate and
self-disable once `install_state.completedAt` is set. Existing installs
that hit the 0.1.16/0.1.17 stuck-at-preflight bug need a fresh
re-install — the broken orchestrator can't self-update past its own
missing-helpers crash.
