## 0.1.22

Operator safety + wizard polish.

### Fixed

**The "already installed" wizard panel's reset SQL was unqualified.**
0.1.15 added a richer panel for the Admin step's 410 (when an admin
already exists in the database). Its third option told the operator
to run `DELETE FROM users; DELETE FROM settings;` to wipe wizard
state and re-run the installer. Problem: `users` and `settings`
are common table names — many apps have them. If an operator's
mysql client is connected to the wrong database when they paste
those statements, they wipe rows from someone else's app.

The panel now:

1. Leads with the **safe** answer (re-install against a different
   database name) — not the destructive one.
2. Demotes the wipe-and-retry path to a clearly-marked rose-tinted
   warning panel ("Only as a last resort — on a TEST database you
   can afford to wipe").
3. Shows the SQL as a one-liner with the database name bound via
   `mysql --database=<your_cavecms_db_name> -e "DELETE …"` — no
   way to accidentally apply it to whatever the operator's mysql
   session was previously connected to.
4. Spells out that `users` and `settings` exist in many apps and
   that running these against the wrong database destroys
   unrecoverable data.

Also fixes the same unsafe form in `instructions.md` (publisher's
operator notes) — same defensive comment + `--database=` binding.

**Template-picker tiles with light backgrounds blended into the
page.** The mockup-preview top half of each tile renders in the
template's own palette. Light palettes (cream backgrounds — Real
Estate, Church, Spa & Wellness, etc.) blended into the wizard's
also-cream page background, making the description below look like
it was floating above an empty gap. Three fixes:

1. Each tile now sits on a `bg-cream-50` background slightly distinct
   from the page bg, so the card itself is visually bounded.
2. A hairline border at the bottom of the mockup-top section gives
   every tile a clear horizontal divide between the palette preview
   and the description copy, regardless of the palette's lightness.
3. Non-selected tiles get a subtle drop-shadow that lifts them off
   the page; hover deepens it. Selected tiles keep their copper-glow.

**Template tiles now show the pointer cursor on hover.** Previously
the `<button>` defaulted to the text cursor since the tile contains
a lot of typography. Explicit `cursor-pointer` matches the click
affordance.

### Migration impact

None. Wizard-copy + tile-styling only.
