## 0.2.3

Installs on shared cPanel hosting no longer fail when the database runs out of connection slots — the installer now clears the way for itself and waits its turn.

### Fixed — install no longer fails with "max_user_connections" on shared hosting

Shared hosts cap how many database connections each account can hold, and a previous copy of the app could be respawned by the host mid-install, silently eating every slot right when the installer needed one. Three fixes work together now:

- During install, the installer parks the web app on a lightweight "this site is being set up" page so nothing competes with the database work, then swaps the real app back in when it finishes.
- If the database still reports its connection slots are full, the installer patiently retries for up to two minutes while they free up, instead of failing on the first refusal — and if it truly can't get in, it tells you exactly what to do next.
- On cPanel, the app now keeps a smaller pool of database connections per instance, sized for shared-hosting limits, so day-to-day running never exhausts the account's quota either.

### Fixed — clean installer output on Node 24

On Node 24 a deprecation warning could print right into the middle of the installer's prompts. The command probes behind it were reworked, so the installer's output stays clean.

### Improved — dashboard update card

The "Update available" card on the dashboard now fades long release notes out gracefully, matching the card's translucent background instead of showing a grey band.
