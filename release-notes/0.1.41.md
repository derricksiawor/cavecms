## 0.1.41

Hot fix on top of 0.1.40 — restores the install path.

### Fixed

**Migration `0023_fix_deleted_at_defaults.sql` wasn't registered in
the drizzle journal.** The 0.1.39 commit added the SQL file but
never appended the matching entry to
`db/migrations/meta/_journal.json`. install-migrate (correctly)
refuses to run when migration files exist on disk but aren't
referenced in the journal — the gate is designed to catch a
half-cut release. The 0.1.40 zip shipped with the same gap, so
every fresh `npx create-cavecms@latest` aborted at step 6/7 with:

```
[install-migrate] ERROR: 1 .sql file(s) not referenced in
db/migrations/meta/_journal.json:
[install-migrate] ERROR:   0023_fix_deleted_at_defaults.sql
[install-migrate] ERROR: The release zip is inconsistent — refusing
to apply migrations.
```

0.1.41 adds the journal entry (idx 23, tag
`0023_fix_deleted_at_defaults`, `breakpoints: true`). Customers
who installed 0.1.39 / 0.1.40 already-installed are unaffected
(their existing journal already has the migration applied or was
created post-0023); the gate only fires during a fresh CLI install.

### Migration impact

None — the migration content is unchanged from 0.1.39. Only the
journal index was missing.
