## 0.2.2

cPanel installs now complete end to end: the app boots, the site renders fully styled, and the setup wizard opens.

### Fixed — the setup wizard opens on cPanel

On cPanel/LiteSpeed the install wizard at `/install` returned a 404. The host runs the app on a private socket, so the middleware couldn't reach its own internal config over local loopback and conservatively assumed the site was already set up. It now defers to a direct database check on that path, so `/install` opens correctly on a fresh install (and still stays hidden once you're set up).

### Fixed — your site loads its styles on cPanel

CSS, fonts, and scripts now load on cPanel. The startup file stages the static assets into the exact folder the bundled server serves them from, so a fresh install renders fully styled instead of as plain HTML.

### Fixed — the app starts on cPanel with no startup-file change

The cPanel startup file stays the default `app.js`, and the installer writes it so cPanel's Node host loads it cleanly — no renaming, no field to change. (Needs Node 20.19+, 22.12+, or 23+, which the installer now checks up front with a clear message.)
