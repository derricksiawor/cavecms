## 0.1.52

Admin login now works over plain HTTP (localhost / LAN installs),
including Safari — plus small polish.

### Admin login bounced to the homepage on HTTP installs

A CLI install runs `NODE_ENV=production` even when it's served over
plain `http://localhost` (a developer testing, a LAN box, a pre-TLS
setup). The session/CSRF cookies were gated on `NODE_ENV`, so they got
the `__Host-` name prefix — and `__Host-` cookies are **required by web
standards to be `Secure`**, which browsers refuse to store over HTTP
(Safari most strictly). The browser dropped the session cookie, the
post-login redirect to `/admin` found no session, and the auth gate
bounced the operator back to `/`.

**Fix:** cookie security now tracks the **request protocol**, not
`NODE_ENV`:

- Cookie names are plain (`cavecms_session`, `cavecms_csrf`, …) — the
  `__Host-` prefix is dropped.
- The `Secure` attribute is set per-request: HTTPS requests (production
  behind nginx / Cloudflare) get `Secure` cookies exactly as before;
  HTTP requests (localhost / LAN / dev) get non-Secure cookies the
  browser will actually store.
- The boot-time guard that hard-required the `__Host-` prefix in
  production is removed.

Every other protection is retained: `httpOnly` on the session cookie,
`Secure` on HTTPS, `SameSite=lax`, host-only (no `Domain`), and the
signed + revocable session JWT. The only thing the `__Host-` prefix
uniquely added was browser-enforced rejection of a `Domain`-scoped
same-named cookie — relevant only to a related-domain attacker who
controls a sibling subdomain on the same parent domain. Single-domain
installs (the norm) have no such exposure; production HTTPS behaviour
in all browsers is unchanged.

### Legal-page titles centered

The Privacy + Terms pages seeded on every install now center their
eyebrow + title heading (body copy stays left-aligned for legal-text
readability).

### AI model picker — dropped the "GA" jargon

The Settings → AI "Inline sparkle" model dropdown no longer shows the
internal "GA" (General Availability) tag. Labels read "fast",
"capable", "newest"; preview models still say "preview" as a genuine
caution.
