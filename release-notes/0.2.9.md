## 0.2.9

Fixes the in-app updater failing to start on 0.2.6 through 0.2.8.

### Fixed — updates run again

On 0.2.6, 0.2.7, and 0.2.8 the update script called one of its own helpers before that helper was loaded, so running an update stopped immediately with a "command not found" error, before anything was downloaded or changed. Your site was never touched, but the update couldn't proceed. This release loads the helper first, so updates run start to finish again. (If your site is on 0.2.6 to 0.2.8 and the updater won't start, this is why; the fix lands the moment you reach 0.2.9.)
