## 0.1.45

The launch release. Premium template polish across every industry
template, a pure-`lx_*` block family (legacy purge), Ed25519 release-
zip signature verification in the in-app updater, and a 3-pillar
review pass closing every LOW-through-CRITICAL defect found across
correctness, security, and quality.

### Templates

Eight industry templates polished end to end (hotel-solenne,
anara-wellness, grace-hill, harbor-and-lane,
ironclad-build, maison-brulee, studio-verde, velocity). Each page
hero opens differently — alignment, overlay, height, ratio rotate
so no two pages look the same. Stock-photo curation swept every
template; clichéd and mismatched photos replaced. Source URLs
upgraded to `w=4000 q=92` and the sharp `lg` variant bumped from
1600px / q82 to 2400px / q88 — `lib/media/sharp.ts` is the root cause
of the "compressed" look on Retina displays. Section templates that
the operator can drop in via the palette (`heroFeaturesCta`,
`heroStatsTestimonial`, `faqAccordion`, `pricingTable`, `contactSection`,
`teamGrid`, `projectShowcase`, `quoteWall`) all rewritten to use
only `lx_*` widgets.

### New lx_ widgets

Ten new luxury composites replace every concept the legacy widget
set covered:
- `lx_testimonial` — portrait + pull-quote + attribution
- `lx_video` — YouTube / Vimeo embed in a cinematic frame, normalised
  to `youtube-nocookie.com` for privacy
- `lx_accordion` — FAQ with native `<details>` semantics
- `lx_tabs` — product-page tabbed sections with proper WAI-ARIA tabs
  pattern (`role="tab"` + `aria-selected` + `aria-controls` +
  keyboard arrow navigation)
- `lx_icon_list` — vertical or grid feature list with lucide icons
- `lx_icon_box` — icon + headline + body card with optional link
- `lx_divider` — hairline rule with optional fleuron section break
- `lx_social_icons` — official simple-icons brand SVGs (16 platforms
  bundled at `/public/icons/social/` under CC0; SVG mask-image so
  one source of truth drives tone)
- `lx_cta_banner` — title + body + primary CTA + optional secondary
- `lx_gallery` — sharp-cornered grid of editorial photos with hover
  captions

### Legacy purge

Every legacy block schema, renderer, palette entry, and dispatch
entry deleted. The codebase ships with only `lx_*` widgets plus
`contact_form` (the one fixed-slot widget the lead route requires).
Migration `0024_legacy_block_type_to_lx.sql` converts 18 legacy
block types to their `lx_*` analog in a single transaction
(per-statement breakpoint), and DELETEs the seven legacy types with
no lx_ home (`hero` / `services_intro` / `featured_projects` /
`about_history` / `stats_row` / `star_rating` / `alert`). Tone enums
are remapped at the SQL layer where the legacy literal isn't valid
in the new schema (`near-black` → `obsidian`, `copper` →
`champagne`); item arrays with renamed fields (icon_list
`{icon,label}` → `{icon,headline}`, social_icons `{platform,url}`
→ `{platform,href}`) are rebuilt via JSON_TABLE + JSON_ARRAYAGG;
nullable optional fields use a JSON_REMOVE pass so a missing
source field doesn't emit JSON null (Zod `.optional()` rejects
null). `FIXED_BLOCK_KEYS_PER_PAGE` trimmed to `contact` only —
home / about / services are block-tree-driven now.

### Security (3-pillar pre-launch review)

**Ed25519 signature verification in the in-app updater (`scripts/cavecms-update.sh`).**
Previously the orchestrator verified only SHA-256 against the same
manifest as the download URL — a compromise of
`cavecms.derricksiawor.com` could ship arbitrary code fleet-wide on
the next "Update now" or post-login warm. The apply route now passes
the manifest's Ed25519 signature + the bundled public key
(`lib/updates/releasePubkey.ts`, single source of truth shared with
the CLI installer) into the script, which verifies via
`node -e crypto.verify` after SHA-256 and BEFORE extract. Refusal
is loud (audit row + dashboard error).

**Apache reverse-proxy template clears inbound X-Real-IP.**
`scripts/apache/cavecms.conf.template` previously cleared
X-Forwarded-* but left X-Real-IP from the client unchanged — and the
Node app reads x-real-ip when the socket peer is loopback (always
true behind ProxyPass). Result: every Apache install bypassed
per-IP rate limits, lockout escalation, maintenance bypass IPs,
and the IP allow/blocklist by rotating an arbitrary X-Real-IP per
request. Now: `RequestHeader unset X-Real-IP` then
`RequestHeader set X-Real-IP "%{REMOTE_ADDR}s"`. Same posture as the
nginx template.

**Login lockout fails CLOSED on DB read error** (`app/api/auth/login/route.ts`).
A transient DB hiccup that threw inside `computeLockState` was
swallowed by `withError`'s 500 path — but only AFTER scrypt verify
ran, giving an attacker an unbounded password-guess window during
DB pressure. Now a wrapped try/catch synthesises a saturated locked
state on read failure and structured-warns. Rate limits remain in
effect throughout.

### Other quality + ops fixes

- **Maintenance toggle moved before Step 3** in `scripts/cavecms-update.sh`.
  Destructive migrations (the `DELETE` pass-2 in 0024) no longer run
  against live traffic.
- **`stopEmailQueueSweeper`** added + wired into the SIGTERM drain so
  pm2 reloads don't generate spurious `email_sweep_failed` log lines.
- **CLI writes `HEALTHZ_TOKEN`** into `env.production` so the
  orchestrator's Step 6 commit-verification healthz probe authenticates.
- **LxTabs WAI-ARIA pattern** corrected (`role="tab"` + `aria-selected`
  + `aria-controls` + arrow/Home/End keyboard nav).
- **LxDivider** dropped the `border-0` + `border-t` Tailwind class
  collision that could render an invisible rule.
- **LxVideo** no longer applies `outerClass` twice (Chunk E spacing
  toolbar values were doubling when a caption was present).
- **Inline-edit threading** added for `LxMap`, `LxVideo`, `LxIconBox`,
  `LxCtaBanner`, `LxCoverImage` overlay fields (eyebrow / title / body),
  plus the dispatcher entries for `LxAccordion` / `LxTabs` /
  `LxIconList` / `LxGallery` (those four still route item-array
  editing through the drawer; the field paths are registered for the
  next pass).
- **LxImagePair AltTextOverlay** now receives the full `data` blob
  so alt-text saves don't 422 due to a missing rightImage MediaRef.
- **LxChannelCard href** inline-editor now uses `kind="url"` (was
  `"plain"`).
- **LxStat** default font is `font-serif` (display tier) to match
  the giant-numeral editorial intent.
- **`readStatus()`** distinguishes ENOENT (silent) from EACCES (logs)
  so a permission-mode bug surfaces in the boot log instead of
  hiding behind a stale "Up to date".
- **Update-checker tick** logs structured warnings on failure (was
  empty catch).
- **`lib/security/escapeHtml.ts`** centralises the HTML-escape
  primitive (middleware was the previous private holder); newsletter
  routes that interpolate into HTML bodies can now reach for the
  shared helper.
- **Migration journal entry 24** registered.
- **Boot-time legacy-block-type audit** (`instrumentation.ts`)
  warns if any `content_blocks.block_type` survives migration 0024
  un-converted. Non-fatal — defense in depth only.

### Migration impact

Migration 0024 runs in a single transaction. The dev's test install
on `test.derricksiawor.com` has not seen the migration yet; its
data is canonical lx_-seeded already, so the conversion pass is a
no-op there. Fresh installs via `npx create-cavecms@0.1.45` never
touch the migration path.

### CLI

`create-cavecms@0.1.45` (synced to release version). The CLI
adds `HEALTHZ_TOKEN` to the sealed `env.production` so the in-app
updater's healthz verification step works on the first update.
Manifest fetch + zip download both verify Ed25519 signatures
against the bundled public key.

### Intentionally deferred

Per the user's "ignore nice-to-haves" filter, the following review
findings are documented and deferred to a follow-up:
- LxEyebrow `prefix: 'rule'` no-op (current renderer is intentional;
  the no-border luxury convention removes the rule prefix).
- LxQuote partial animate vs whole-stack animate (cosmetic
  inconsistency).
- Per-item inline-edit for `lx_accordion` / `lx_tabs` / `lx_icon_list`
  / `lx_gallery` (operators edit array items via the EditDrawer;
  paths are registered for the future overlay implementation).
- `mergeFieldValues` silent drop of unrecognised paths (defensive
  polish — would improve operator feedback on AI-suggested edits).
- Tarball-mode expand/contract migration guard (`git diff` doesn't
  work without a `.git` dir; relevant only when a future migration
  adds destructive schema changes).
- `CAVECMS_RELEASE_MANIFEST_URL` / `CAVECMS_RELEASE_DOWNLOAD_ORIGINS`
  origin allowlists (the Ed25519 signature is the actual defense;
  these would be a hardening layer).
