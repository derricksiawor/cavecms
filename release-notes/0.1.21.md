## 0.1.21

Two hotfixes + one UX win.

### Fixed

**Logo upload 500'd with "sharp module not found" on Linux servers.**
Sharp is a native module — its binary varies per OS + CPU. The
publisher's release-build runs on macOS, so the standalone bundle
shipped with sharp's darwin-arm64 binary instead of linux-x64.
At runtime on the customer's Linux box, sharp's native loader
couldn't find the platform-matching `.node` file and threw at
`POST /api/install/logo`'s first image-resize call.

The release build now runs `npm install --os=linux --cpu=x64
--no-save sharp@0.33.5` inside `.next/standalone/` AFTER `next
build`, so the zip carries linux-x64 binaries alongside whatever
the publisher's local platform installed. Customers on Linux see
sharp resolve cleanly; the publisher's local `pnpm dev` workflow
is unchanged (still uses the darwin binary).

### Added

**Back button on every install-wizard step.** Operators who want
to change a previously-picked template, swap a logo, or just
verify what they entered can now hit a small ← Back affordance
above each step's header. Welcome and Done correctly omit Back
(no predecessor / terminal). The Admin step's "already installed"
panel also omits Back — it's terminal for that database.

* `StepHeader` gained an `onBack` prop and a `backDisabled` prop
  (greys the affordance while a POST is in flight — closes the
  race where clicking Back mid-submit would unmount the step but
  let the request still mutate server state).
* `TemplateStep` now restores the previously-picked tile when the
  operator navigates Back into it — the slug is lifted to the
  wizard parent (`pickedTemplateSlug` state) so it survives the
  AnimatePresence unmount.
* The Back affordance is keyboard-focusable (`<button type="button">`
  with visible focus ring + `aria-label="Go back to the previous
  step"`); hit target is ≥28px per WCAG 2.5.5.

### Migration impact

None. Re-install or in-app update from 0.1.20 to 0.1.21 picks up
both fixes. No DB or config changes.
