## 0.2.4

A reliability release for cPanel hosting and backups: your custom login address now always works, updates run on cPanel, failed updates explain themselves, and backups tell you what's happening from the first click.

### Fixed — your custom login address works on cPanel

Choosing a custom login address (in the setup wizard or under Settings → Security) could lock you out on cPanel hosting: the new address showed a 404, and so did the old one. CaveCMS now keeps every copy of that setting in step automatically — including the one the routing layer reads — and applies it on the spot. No more hand-editing files on the server.

### Fixed — updates work on cPanel

Clicking Update now on cPanel hosting failed immediately at "Getting ready". The health check behind it was knocking on a door the host doesn't expose; it now reaches the app the way cPanel actually serves it, so updates proceed normally.

### New — see why an update failed

When an update fails, the progress window now has a "Show technical details" link with the logs from the failed run — so you (or whoever helps you) can see exactly what happened without needing shell access to the server.

### Fixed — backups tell you what's happening

- Clicking Back up now opens the progress window immediately — no more silent wait before anything appears.
- Uploading a backup file to restore from now shows a live progress bar while it uploads.
- Large backup uploads no longer stall partway on shared hosting — the upload streams straight to disk instead of being held in memory.
- If a cloud backup needs an encryption passphrase before it can include your secret keys, the message now says exactly that — and what to do — instead of a generic "try again".
- If backup encryption is configured but the encryption tool is missing from the server, the backup now fails with a clear message instead of quietly saving an unencrypted archive.
- Backup results are reliably recorded in the activity log again (a shell quirk silently dropped them).

### Improved — safer restore uploads

Uploaded restore files are checked the moment they arrive: anything that isn't a genuine backup archive is rejected on the spot, with a specific message for encrypted archives that need the command line.

### Improved — dashboard update card

The "Update available" card on the dashboard now previews long release notes the same way the Updates page does — a gentle fade with a Show more link, sized to the card.
