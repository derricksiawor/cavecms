## 0.1.40

Page 404 fix for every CMS-slug page + six more image-bearing templates +
a Cloudflare-style DNS step in the get-started docs.

### Fixed

**Reverse-proxy page 404s.** Every non-home CMS slug (`/rooms`,
`/dining`, `/story`, `/reservations`, `/thank-you-enquiry`, every
operator-authored page) returned 404 on 0.1.38/0.1.39 despite valid
DB rows and a clean middleware rewrite. Root cause: Next.js's config
schema silently strips `experimental.trustHostHeader` from
`next.config.ts` during build because the key isn't in the public
schema, even when cast through `NextConfig['experimental']`. The
bundled `.next/standalone/server.js` ends up with
`"trustHostHeader":false` baked in. Without that flag, resolve-routes
builds `initUrl` from the bound listener address
(`https://localhost:8201`) instead of the request's Host header.
Middleware's `NextResponse.rewrite` produces a URL with the public
Host as origin → origins mismatch → router-server treats it as an
external rewrite → fires an internal HTTP proxy that loops out
through the public reverse proxy and re-enters middleware on
`/_page/<slug>` → `refuseInternalPageRoute` returns 404. Fixed with a
new `scripts/postbuild-patch-trust-host-header.ts` post-build step
that mutates the bundled `server.js` to flip the flag to `true`.
Idempotent, fails loudly if the embedded JSON shape changes upstream,
no-op on non-standalone builds.

### Added

**Six more image-bearing templates.** grace-hill (church),
harbor-and-lane (real-estate brokerage), ironclad-build (contractor),
maison-brulee (restaurant), studio-verde (creative studio), velocity
(SaaS) all now wired through the image-bearing widget helpers
introduced in 0.1.38. Each template ships 12 curated stock photos
(72 total, 388 variant files, ~40 MB shipping variants) downloaded +
processed by the existing `scripts/release/build-template-media.ts`
pipeline at release-build time. Combined with the hotel-solenne and
anara-wellness templates from 0.1.38, all 8 industry templates now
seed with cover imagery, hero shots, and figure widgets out of the
box — operators picking any of them get a complete visual experience
on first wizard-walk.

### Migration impact

None — runtime config patch + template content. No schema changes.
`postbuild` script chain extended; the patcher runs first, then the
existing slug-collision check.
