## 0.1.18

Second hotfix for the in-app updater on CLI-installed sites.

### Fixed

**Update preflight failed with "scripts/lib/cavecms-update-helpers.sh:
No such file or directory".** 0.1.16 fixed the status-path 500 but
hit a deeper packaging bug: the apply route resolves the orchestrator
shell script relative to `process.cwd()` — which is `.next/standalone/`
in production. Next.js's standalone tracer sees the literal string
path `scripts/cavecms-update.sh` in the apply route and copies that
ONE file into the standalone bundle, but it has no way to follow
shell-`source` declarations, so `scripts/lib/cavecms-update-helpers.sh`
(sourced by the orchestrator at line 137) is missing from the
standalone bundle. Result: every update stalled at "Preparing your
data" because the orchestrator died on its very first `source` line.

The release build now copies the full `scripts/` subtree into both
the install root AND `.next/standalone/scripts/` so the orchestrator
finds its helpers no matter which cwd it resolves from.

### Migration impact

None for existing installs that already managed to update past
0.1.15. Customers stuck on 0.1.16 / 0.1.17 with this bug should
re-install from 0.1.18+ — the orchestrator can't self-update past
its own missing-helpers crash.
