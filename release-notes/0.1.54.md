## 0.1.54

The Featured Projects block is back — as a first-class `lx_` block that
auto-renders your featured work — plus a sweep that removes the last
references to the purged legacy block family.

### Featured Projects block

`lx_featured_projects` is a card grid that shows the projects you mark
**Featured** (Projects → Featured order), pulled live: each card resolves
its name, tagline, and hero photo straight from the project, so renaming a
project, swapping its hero, or re-ordering Featured updates the grid
automatically.

- **No per-block setup:** there's nothing to wire up on the block itself —
  it auto-renders your Featured projects in their Featured order (capped at
  12). Curate the set in one place: the Projects admin.
- **Drawer controls:** optional section heading, column count (2/3/4),
  tone, and scroll animation.
- **Empty state:** with no project marked Featured, the grid renders
  nothing on the public page and shows an inline hint in the editor
  pointing you to Projects → Featured order.
- **Migration:** any legacy `featured_projects` row on an install that
  upgrades through `0024` is now **converted** to `lx_featured_projects`
  instead of being deleted — its heading is preserved and the grid picks
  up the install's Featured projects. (Pre-0.1.54, that migration deleted
  the block outright.)

### Legacy cleanup

Removed the remaining vestigial references to the purged legacy block
types (hero / services_intro / about_history / featured_projects / image /
gallery / stats_row, etc.) across the editor, the AI eligibility map, the
admin page-list excerpts, the block-search guard, and the new-page clone
previews. Behaviour is unchanged for current blocks; the dead branches and
stale comments are gone, and the palette/guard tests now assert the lx_
catalog explicitly.

Also corrected stale integration-test fixtures + assertions that were
left at legacy block types after the lx_ migration.
