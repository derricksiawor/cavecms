## 0.1.26

Fourth in-app updater hotfix on CLI-installed sites.

### Fixed

**Update preflight failed with "Your site isn't responding properly"
at step 1.** The orchestrator's healthz probe defaults to
`http://127.0.0.1:3040/healthz` (the bare-metal `deploy.sh` layout's
hardcoded port). But CLI installs run on whatever port the operator
chose — 8201 on the test install, customer-chosen on real installs.
The probe hit port 3040, got `Connection refused`, and the modal
failed at "Getting ready" before any actual update work could begin.

Fixed in `app/api/admin/updates/apply/route.ts`'s `buildScriptEnv`:
when `CAVECMS_HEALTHZ_URL` isn't explicitly set, derive it from
`process.env.PORT` (which IS set per-install by the CLI). The
orchestrator inherits the correct URL and polls the actual running
port.

### Migration impact

Customers re-installing from 0.1.26+ get the fixed apply route in
their standalone tree. Customers stuck on 0.1.16-0.1.25 installs
that hit this stall need a fresh re-install.
