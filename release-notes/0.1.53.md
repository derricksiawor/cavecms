## 0.1.53

Footer theme is now configurable, matching the header.

### Footer theme setting

The footer was hardcoded to a single dark (near-black) palette. It now
has its own **Settings → Footer → Footer theme** picker with the same
five options as the header — Obsidian (dark, default), Cream, Ivory,
Champagne, Bone.

- **Independent setting:** footer theme is separate from the header
  theme — change either on its own at any time.
- **Linked at setup:** during the install wizard (template step + the
  branding step), the chosen theme seeds BOTH the header and the footer
  to the same value, so a freshly-installed site has a matching header
  and footer out of the box. After install they're fully independent.
- **No change to existing sites:** the default `obsidian` footer theme
  reproduces the previous near-black footer byte-for-byte; existing
  installs render identically until someone picks a different footer
  theme.

Implementation mirrors the header: a `lib/cms/footerTheme.ts` class-set
map (surface, logo filter, text tiers, copper accent, link/hover,
border) consumed by `SiteFooter`. Light themes correctly flip the
surface to light, text to dark, keep the copper accents readable, and
drop the dark-logo invert. Verified in-browser on the Cream theme
(cream surface, near-black text, copper-700 headings — all legible).
