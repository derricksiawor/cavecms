### Architectural changes

- New image-bearing helpers in `lib/cms/siteTemplates/_shared.ts`:
  `coverImage`, `figure`, `imagePair`, `heroWithImage`, `imageCardsRow`,
  `imageWithText`. Each emits a `WidgetSpec` whose `_imageKeys` +
  `_imageAlts` siblings transport unresolved image references to the
  install route resolver.
- New `lib/cms/siteTemplates/extractImageKeys.ts` walker â€” single source
  of truth used by the prebuild validator + the install route's
  pre-flight manifest cross-check.
- New `scripts/release/build-template-media.ts` â€” at release-build time,
  downloads originals declared in each template's `media-sources.json`,
  runs the same `processToVariants` pipeline the upload route uses (q82
  webp + q86 jpg og), and emits per-template manifests at
  `dist/template-media/<slug>/`. `release:build` copies the tree (minus
  the originals subdirs) into the staged install at
  `.next/standalone/template-media/`.
- `app/api/install/template/route.ts` now reads the bundled manifest
  pre-flight (fails 422 `template_media_key_missing` if any key is
  unresolved), copies variant files with fresh UUIDs into
  `UPLOADS_ROOT/variants/` inside the seed transaction, INSERTs `media`
  rows + reverse-index `media_references` rows, and INJECTs the resolved
  MediaRef into every widget's `data` field BEFORE `parseAndSanitize`
  fires. On TX rollback every copied file is unlinked so failed seeds
  don't orphan variants on disk.
- `scripts/validate-site-templates.ts` now fails the prebuild if any
  imageKey a template references isn't covered by its `media-sources.json`.

### Migration impact

hotel-solenne / anara-wellness from the wizard) receive imagery.
