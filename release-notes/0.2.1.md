## 0.2.1

cPanel installs now start reliably, and reinstalling is safe.

### Fixed — your site starts on cPanel / LiteSpeed

The cPanel startup file is now `app.cjs`, which the cPanel Node host (LiteSpeed's loader or Passenger) can load with `require()` without the module error that previously left sites showing a 503. After installing, set your app's "Application startup file" to `app.cjs` and restart. (The installer writes it and prints this step for you.)

### Improved — reinstalling never deletes your files

Reinstalling into a cPanel app folder now MOVES any leftover scaffolding aside into a timestamped, fully recoverable backup. It never deletes. And it refuses outright to touch a folder that holds a live install's data (your sealed env file or your uploads), a web root like public_html, or any files CaveCMS didn't create, listing exactly what's there if it stops. So a mistaken run can't wipe your site.

### Improved — one less question on cPanel

The installer no longer asks for an "App listen port" on cPanel. The host assigns it.
