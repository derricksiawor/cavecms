## 0.1.15

UX improvement to the install wizard's already-installed guard.

### Fixed

**"This install is already set up. Refresh to go to login." was the
wrong message.** When the install wizard's Admin step returned 410
(an admin already exists for the database), the wizard collapsed the
state into a one-line error telling the operator to refresh and go
to login. That copy assumed the operator hit the wizard URL by
mistake and meant to sign in — but the actual scenario is more often
the opposite: the operator was trying to install a new site and ran
into a database that already had an admin from a previous attempt.

Replaced the one-liner with an info panel that explains what
happened and offers three concrete actions:

* If you remember the admin credentials → sign in at the
  `LOGIN_PATH` value from `env.production`.
* If you forgot the password → your server admin can reset it with
  the bundled reset script.
* If you wanted a fresh install → clear the wizard rows
  (`DELETE FROM users; DELETE FROM settings;` for test envs, or
  point at a different database for production) and re-run the
  installer.

### Migration impact

None. Wizard-copy change only.
