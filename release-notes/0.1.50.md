## 0.1.50

Contact form is now surface-aware. Two templates (harbor-and-lane,
anara-wellness) shipped a contact form on a cream/ivory section. The
form's input fill + label + heading were hardcoded to dark-theme
tokens (`text-ivory` / `bg-ivory/5` / `text-champagne`), so the entire
form became invisible against the light section bg — labels readable,
input boxes vanished, the page looked half-built.

### Fix — no hardcoded theme tokens anywhere

The root cause was that the renderer made an assumption about the
surface instead of reading the surface. Fixed at the architecture
level:

- The widget render dispatcher (`components/blocks/index.tsx`) now
  receives the parent section's `meta` and threads it into the
  per-widget `BlockRendererArgs`. Both the public-render tree
  (`BlockTreeRenderer`) and the editable tree
  (`EditableBlockTreeRenderer`) pass the parsed `SectionMeta` down.
- `components/blocks/ContactForm/render.tsx` calls
  `isSectionSurfaceDark(sectionMeta)` to decide between dark-theme
  tokens (ivory / champagne — used against `obsidian`, `near-black`,
  cover-photo + dark-overlay heroes) and light-theme tokens
  (obsidian / copper-700 — used against `ivory`, `cream`, `champagne`,
  `bone`).
- `components/leads/ContactForm.tsx` takes a `surface: 'dark' | 'light'`
  prop and switches three constant sets per surface: `INPUT_BASE`,
  `LABEL_TEXT`, the native date/time `[color-scheme]` hint. Default
  stays `'dark'` so any external caller still in the wild renders
  correctly on obsidian.

Result: drop a `contact_form` block on any section in any template,
the form picks up readable tokens automatically. No template seed
needs to declare a theme. No future renderer rewrite is needed when
a template lands on a new surface colour.

### Affected templates (no seed changes needed)

- `harbor-and-lane/contact` (on `bg: 'ivory'`) — now renders with
  obsidian inputs on the cream surface.
- `anara-wellness/booking` (on `bg: 'cream'`) — same fix carries.
- Every other template's contact form sits on a dark section and
  reads unchanged.
