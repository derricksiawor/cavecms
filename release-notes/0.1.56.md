## 0.1.56

Installs + updates now work from cloud servers behind Cloudflare, the
empty-page widget insert actually works, the admin page editor's quick-add
nests like the inline editor does, and the icon box's style options are
editable from its drawer.

### Installs + updates work behind Cloudflare Bot Fight Mode

The installer and the in-app updater fetched the release host with curl and
Node's `fetch`. Cloudflare Bot Fight Mode (on the Free plan, where it can't be
scoped to specific paths) fingerprints those clients by their TLS handshake
and returns **403** from datacenter IPs — so `npx create-cavecms` and the
update checker silently failed on most cloud VPSs (DigitalOcean, AWS, Hetzner,
Linode…), even though they worked from un-flagged IPs. Every release-host
download — the installer's manifest + zip, the update check, and the updater's
release download — now goes through `wget` with a browser User-Agent, the one
client/UA combination that clears the challenge. A custom `X-CaveCMS-Client`
header (which the bot check ignores) keeps the requests identifiable in origin
logs. `wget` is required on the server; the installer fails early with a clear
message if it's missing.

### Adding a widget to an empty page now works (0.1.55 regression fix)

0.1.55 introduced the section→column→widget guard for empty pages, but the
auto-created section was posted without its required settings — the server
rejected it, so clicking a starter block on a blank page silently did
nothing. The auto-wrap now sends a valid default section (one column, cream
background, standard padding), so the first block lands correctly inside a
section and column. If the widget itself can't be saved after the section is
created, the empty section is rolled back so nothing is left stranded.

### The admin page editor's "Add block" nests properly

Quick-adds from the **Pages → editor → Add block** menu used to drop the new
block loose at the top of the page instead of inside a section and column.
Every quick-add now builds the same section→column→widget structure the
public-page inline editor produces (with the same rollback safety), so blocks
added from either surface stay consistent and reorderable.

### Icon box style options are editable from the drawer

The icon box exposed accent, icon, alignment, tone, and animation in its
schema but offered no way to change them from the edit drawer — only the
headline, body, and link were reachable. The drawer's Style tab now renders
all of them: a Lucide icon picker, the accent style (champagne outline /
fill / cream tint), alignment, text tone, and scroll animation.
