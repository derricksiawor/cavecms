## 0.1.29

Fifth (and hopefully final) in-app updater hotfix.

### Fixed

**Update preflight false-failed with "Database isn't MariaDB" on
Ubuntu's MariaDB build.** The orchestrator's preflight queried
`@@version_comment` to discriminate MariaDB from MySQL — but distro
packages overwrite that field with the OS string. On Ubuntu's
MariaDB build, `@@version_comment` returns "Ubuntu 22.04", which
didn't match the "MariaDB" pattern and the preflight halted at
step 1.

`VERSION()` is the reliable signal: legitimate MariaDB always emits
"X.Y.Z-MariaDB-..." (e.g. `10.6.22-MariaDB-0ubuntu0.22.04.1`); pure
MySQL never includes that suffix. The fix in `scripts/cavecms-update.sh`
matches `VERSION()` for `*MariaDB*` instead of `@@version_comment`.
The bare-metal preflight (`scripts/preflight.sh`) already used the
correct VERSION()-based primary check — this aligns the in-app
updater with it.

### Migration impact

Customers on 0.1.27/0.1.28 installs hit by the false-fail need a
fresh `npx create-cavecms@latest` re-install to pick up the 0.1.29
orchestrator. Subsequent updates work in-app.
