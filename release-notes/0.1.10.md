## 0.1.10

Cleanup release that fixes UX rough edges surfaced during the first
end-to-end customer-journey test on test.derricksiawor.com.

### Fixed

**Wizard welcome step — operator-facing copy.** Bullet labels for
the 4 optional steps (Branding, Contact, Email, Security) said
"skippable" — replaced with "(Optional)" which reads cleaner and
matches the wizard's own "Step N — optional" eyebrow on each step.

**Bare `/` redirect leaks loopback host in Location header.** When a
fresh install was redirecting the public site root to `/install`,
the middleware built the redirect with `new URL('/install', req.url)` —
which resolved against `req.url`'s host (the loopback upstream
`http://localhost:8201/...` per `HOSTNAME` env in PM2). The browser
got `Location: https://localhost:8201/install`, totally unreachable.
Switched to emitting a relative `Location: /install` — browsers
resolve relative redirects against the URL the user typed, so the
public hostname stays intact regardless of how the upstream is bound.

**One-pager install ships clean nav + footer.** The settings-registry
defaults for `site_header.navItems` previously included
`Home / Projects / Services / About / Contact`. None of those pages
were populated with operator content on a fresh install, so RSC
prefetches to `/services` + `/about` 500'd in the browser. Same
problem in the footer default — a "Legal" column with `/privacy` +
`/terms` links pointing at empty pages, plus a `legalLinks` strip
with the same. Defaults now ship empty: `navItems: []`,
`primaryCta: null`, `footer.columns: []`, `footer.legalLinks: []`.
Operator wires up their own links under Settings → Branding /
Settings → Footer after the install. Schema change: `primaryCta`
on `siteHeader` is now nullable (the public renderer already had a
`hasCta &&` short-circuit, so this is shape-only).

### Known limitations (deferred — surfaced during the test)

**Cloudflare auto-injects `cdn-cgi/scripts/email-decode.min.js` when
the site has `mailto:` links + Email Address Obfuscation is enabled.**
Our `script-src 'nonce-…' 'strict-dynamic'` CSP rejects it (no
nonce), browser logs a CSP violation. Cosmetic — the obfuscation
script isn't needed for our own email rendering. Recommended fix for
operators behind Cloudflare: zone settings → Scrape Shield → disable
Email Address Obfuscation. We don't ship a server-side opt-out
because the toggle is per-zone, not per-request. Will document in
the install wizard's contact-email step in a follow-up.

**Minified React error #418 (hydration text mismatch) on the admin
dashboard.** Visible in the browser console; the UI itself works.
Likely a locale-dependent timestamp render or a server-vs-client
divergent value in the admin chrome. Requires `pnpm dev` repro to
read the unminified error and pin the exact component — deferred to
the next release.

### Migration impact

None. All changes are application-level (registry defaults +
middleware + wizard copy). Existing installs that already wrote
`site_header` / `footer` rows to the DB keep their current values —
the new defaults only apply when a row is missing OR on a fresh
install. Operators who want the new clean defaults on an existing
install can clear those two `settings` rows from the dashboard or
via mysql.
